# ストリート野球

## 開発環境
- Godot 4.6

## ゲーム概要
`ストリート野球` は、ヤンキーや一般人による独自野球リーグを舞台にした球団運営シミュレーションです。
ルールや年間の流れは NPB 準拠ベースを土台にしつつ、ところどころにストリート感や遊び心のあるイベントを入れていく方針です。

プレイヤーは担当球団を選び、その球団を中心に
- 日々の進行
- 編成管理
- 試合観戦と采配
- 球団運営
を進めていきます。

## 現在の基本方針
- まずはゲーム全体が破綻なく回ることを優先する
- 細かいバランス調整や演出強化は後回しにする
- 画面や機能は後で分離・整理できる前提で作る
- ライブ試合は本命、過去試合は情報確認用に分ける

## 現在のコアループ
1. 担当球団を選ぶ
2. ホーム画面で日付を進める
3. 必要なら当日のライブ試合を見る
4. チーム管理や球団運営を行う
5. シーズンを進め、年をまたいで複数年遊ぶ

## 日程とシーズン進行
- 2026年開始
- 日付ベースで進行
- 年間進行を前提にした構造
- 公式戦は NPB 準拠ベースの 140 試合前後を目標にした土台
- 開幕前 / シーズン中 / オフシーズン を日付から判定
- 年末を越えると自動で次年度へ移行

## 画面構成
### 1. TeamSelect
- 担当球団選択画面
- 新規開始時の入口

### 2. LeagueHome
- ホーム画面
- 日付、担当球団、進行ボタン、今日の予定、昨日の結果、ニュース、簡易速報を表示
- `今日の試合へ` からライブ試合へ入る

### 3. TeamManagement
- 編成管理画面
- 対右 / 対左打線
- 先発ローテ
- ブルペン
- チーム方針

### 4. LeagueInfo
- リーグ全体の確認画面
- 順位表
- 個人成績
- 年間日程
- シーズン履歴
- ニュース

### 5. RosterView / PlayerDetail
- 担当球団の選手一覧と個別詳細
- 年齢、年俸、能力、成績、起用状況などを表示

### 6. GameDetail
- 過去試合の情報確認画面
- スコア、勝敗投手、詳細ログを表示
- 過去試合の本格リプレイは行わない方針

### 7. MatchScene
- 当日の担当球団ライブ試合専用画面
- 一球ごと / 一打席ごとの進行
- オート再生
- オート速度切り替え
- 停止場面設定
- 采配と指示

### 8. 球団運営系画面
- FrontOffice: ハブ画面
- ContractOffice: 契約更改と FA 導線
- FAMarket: FA候補一覧
- FacilityOffice: 施設強化
- SponsorOffice: スポンサー営業
- StaffOffice: スタッフ管理
- ScoutDraftOffice: スカウト / ドラフト候補の受け皿

## ライブ試合方針
- MatchScene は当日の担当球団ライブ試合専用にする
- 過去試合は GameDetail で確認する
- ライブ試合は一発勝負の考え方を優先する
- 見たい時だけライブに入り、見ない時は自動進行で回せるようにする

### 現在入っている機能
- 一球ごと / 一打席ごとの切り替え
- オート再生
- 低速 / 標準 / 高速 の速度切り替え
- 停止なし / チャンス / ピンチ / 重要場面 の停止設定
- 強攻 / バント / 進塁重視 / 敬遠 の指示
- 球種、コース、カウント、球数の表示
- 試合確定後の結果反映と試合後サマリー

### 試合実装の考え方
- 試合ロジックと演出を分ける
- まず内部で試合イベント列を作る
- そのイベントを画面で順に再生する
- 最初はテキストやUI中心で進め、後からアニメや演出を足す

### 指示の影響方針
試合中の指示は `能力 + 状況 + 指示` を合わせて結果分布を動かす方針です。
影響先は主に以下です。
- 打席結果の確率
- 進塁処理
- 球数や一球イベント
- 起用や試合運び

現時点では、まず `指示で結果分布が変わる構造` を優先し、あとで微調整していきます。

## 長期プレイ方針
- 複数年プレイを前提にする
- シーズン履歴を残す
- 通算成績、順位、優勝回数を追えるようにする
- 人気、予算、年俸総額の推移を見られるようにする

## 球団運営方針
- 球団運営はハブ画面として扱う
- `契約・FA` `施設` `スポンサー` `スタッフ` `スカウト・ドラフト` に分けて育てていく
- まずは最低限の機能を置き、後から各ページを強化する

### 現在入っている球団運営要素
- 施設強化
- スポンサー営業
- スタッフ増減
- 契約切れ間近の選手確認
- 簡易契約更改
- FA候補一覧の閲覧
- スカウト / ドラフト候補の受け皿

### 今後追加したい経営要素
- 移籍 / FA交渉
- トレード
- 選手との交渉
- スカウト割り振り
- ドラフト本体
- スポンサー契約の拡張
- スタッフの役割最適化
- 経営イベント

## 現在の優先方針
- まずは全体を完成させる
- 各機能は最低限でも繋げる
- 中身の充実、微調整、演出の強化はあとで行う
