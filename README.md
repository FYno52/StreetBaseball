# ストリート野球

## 開発環境
- Godot 4.6

## ゲーム概要
`ストリート野球` は、ヤンキーや一般人による独自野球リーグを舞台にした球団運営シミュレーションです。
ルールや年間進行は NPB 準拠ベースを土台にしつつ、ところどころにストリート感や遊び心のあるイベントを入れていく方針です。

プレイヤーは担当球団を選び、その球団を中心に以下を進めます。
- 日々の進行と試合消化
- 編成管理
- ライブ試合での采配
- 球団運営
- 複数年プレイ

## 現在の大きな方針
- まずはゲーム全体の型を完成させる
- 細かい微調整や数値バランスは後回しにする
- ライブ試合は本命、過去試合は情報確認に寄せる
- 365日進行のカレンダーを土台にする

## コアループ
1. 担当球団を選ぶ
2. ホーム画面で日付と今日の予定を確認する
3. 必要ならチーム管理・球団運営を行う
4. 試合を見るか、自動進行で進める
5. シーズンを越えて複数年運営する

## 画面構成
### TeamSelect
- 担当球団選択画面

### LeagueHome
- ホーム画面
- 日付、担当球団、進行ボタン、今日の予定、昨日の結果、ニュース、簡易速報を表示
- `今日の試合へ` からライブ試合へ入る

### TeamManagement
- 編成管理画面
- 打線、ローテ、ブルペン、方針を管理

### LeagueInfo
- リーグ全体の情報画面
- 順位、個人成績、年間日程、ニュース、試合詳細を確認

### CalendarScene
- 年間カレンダー画面
- 今日の年間イベント
- 次に来るイベント
- 月ごとに切り替えて、その月のイベントと担当球団予定を確認
- 担当球団の年間予定
- 年間イベント一覧

### RosterView / PlayerDetail
- 選手一覧と選手詳細
- 年齢、年俸、能力、起用状況、成績などを表示

### GameDetail
- 過去試合の詳細確認画面
- スコア、勝敗投手、詳細ログを表示

### MatchScene
- 当日の担当球団ライブ試合画面
- 一球ごと / 一打席ごと進行
- オート速度切替
- 停止場面設定
- 采配・指示
- 試合後サマリー

### 球団運営系画面
- FrontOffice: ハブ画面
- ContractOffice: 契約更改と FA 導線
- FAMarket: FA 候補一覧
- FacilityOffice: 施設強化
- SponsorOffice: スポンサー営業
- StaffOffice: スタッフ調整
- ScoutDraftOffice: スカウト / ドラフト候補の受け皿

## 試合実装の考え方
### 基本方針
- 試合ロジックと演出を分ける
- まず内部で試合イベント列を作る
- そのイベントを画面で順に再生する
- 最初は試合の型を完成させ、後から演出を足す

### イベントの粒度
- まずは打席単位
- 次に一球ごとの `pitch` イベントを入れる
- 最終的にはライブ采配で結果分布を変える

### 指示の影響先
試合中の指示は `能力 + 状況 + 指示` を合わせて結果分布を動かす前提です。
主な影響先は以下です。
- 打席結果の確率
- 進塁処理
- 球数や一球イベント
- 起用や試合運び

例:
- `強攻`: 長打と三振が増えやすい
- `バント`: バント専用結果に寄せる
- `進塁重視`: 進塁打や単打寄りにする
- `敬遠`: 四球寄りにする

### 試合画面の役割分離
- 過去試合は `GameDetail` で確認する
- ライブ試合は `MatchScene` に集約する
- `MatchScene` は一発勝負の前提で、巻き戻しや再入場で結果を変えられないようにする

## 365日進行と年間イベント
### 基本方針
- 1月1日から年末まで日付を進める
- 開幕日になると自然に公式戦へ入る
- シーズン終了後はそのままオフシーズンへ移る
- 年末を越えると次年度へ進む

### 年間イベントの土台
球団運営要素を成立させるために、日付に応じた年間イベントを持たせる。
まずは以下を想定する。
- 春季キャンプ
- オープン戦期間
- 開幕日
- ドラフト準備期間
- ドラフト会議
- 契約更改期間
- FA交渉期間
- スポンサー更改期間
- スタッフ見直し期間

### 実装方針
- `LeagueState` が「今日の年間イベント」を返す
- ホーム画面で `今日は何の日か` を表示する
- 契約更改や FA、ドラフト系機能は、その期間だけ有効化する

## ホーム画面の方向性
ホーム画面は最終的に、パワプロのペナントや栄冠ナインのような `カレンダー感のある中心画面` に寄せる。
まずは以下を1画面で把握できる形を目指す。
- 今日の日付
- 今日の年間イベント
- 次に来るイベント
- 今日の試合予定
- 担当球団の近況
- 昨日の結果
- ニュース
- 進行ボタン
